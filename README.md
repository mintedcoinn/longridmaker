# LongridMaker

LongridMaker превращает учебные материалы в черновик лонгрида и SCORM 1.2-пакет для LMS/Moodle. Приложение принимает файл лекции, извлекает текст, таблицы и изображения, собирает структурированные блоки, добавляет мини-квиз и позволяет скачать готовый ZIP-пакет SCORM.

## Возможности

- загрузка учебных файлов через веб-интерфейс;
- разбор PDF, PowerPoint, Word, TXT и аудио/видео файлов;
- генерация структуры лонгрида: разделы, лиды, абзацы, ключевые термины, выводы;
- извлечение таблиц и изображений из исходных материалов;
- мини-квиз по содержанию;
- экспорт результата в SCORM 1.2 ZIP-пакет.

## Состав проекта

```text
api/      Backend на FastAPI: парсеры, pipeline, экспорт SCORM.
ui/       Frontend на React + Vite + MUI.
scorm/    Каталог для SCORM-связанных материалов.
docs/     Дополнительная документация проекта.
```

## Требования

- Python 3.10 или новее.
- Node.js 20 или новее.
- npm.
- Для аудио-транскрибации: ключ OpenAI API.
- Для LLM-улучшения структуры лонгрида: ключ DeepSeek API. Без него pipeline работает в эвристическом режиме.

## Быстрый запуск

Откройте две командные строки из корня проекта.

### 1. Backend

```cmd
cd api
python -m venv .venv
.venv\Scripts\activate.bat
pip install -r requirements.txt
python -m uvicorn app.main:app --reload
```

Backend будет доступен на `http://127.0.0.1:8000`.

Полезные страницы:

- `http://127.0.0.1:8000/docs` - Swagger UI;
- `http://127.0.0.1:8000/formats` - список поддерживаемых форматов;
- `http://127.0.0.1:8000/llm/status` - текущая LLM-конфигурация.

### 2. Frontend

```cmd
cd ui
npm install
npm run dev
```

## Переменные окружения

Переменные задаются в терминале перед запуском backend.

### Генерация блоков через DeepSeek

Если переменные не заданы, приложение использует эвристический режим без внешней LLM.

```cmd
set DEEPSEEK_API_KEY=ваш_ключ
```

### Распознавание аудио

Аудиофайлы требуют OpenAI API key.

```cmd
set OPENAI_API_KEY=ваш_ключ
set OPENAI_WHISPER_MODEL=whisper-1
```

Дополнительные параметры:

- `OPENAI_TRANSCRIPTION_LANGUAGE` - язык аудио, например `ru`;
- `OPENAI_TRANSCRIPTION_PROMPT` - подсказка для распознавания;
- `OPENAI_TRANSCRIPTION_MAX_BYTES` - максимальный размер аудиофайла в байтах, по умолчанию 25 МБ.

## Поддерживаемые форматы

Основные документы:

- `.pdf`
- `.pptx`
- `.pptm`
- `.docx`
- `.txt`

Аудио и видео для транскрибации:

- `.mp3`
- `.mp4`
- `.mpeg`
- `.mpga`
- `.m4a`
- `.wav`
- `.webm`

Актуальный список можно получить через endpoint `/formats`.

## Эксплуатация через интерфейс

1. Запустите backend и frontend.
2. Откройте frontend в браузере.
3. Нажмите `Выбрать файл` и загрузите учебный материал.
4. Нажмите `Собрать лонгрид`.
5. Проверьте оглавление, текстовые блоки, таблицы, изображения и мини-квиз.
6. Нажмите `Скачать SCORM`, чтобы получить ZIP-пакет.
7. Загрузите ZIP в Moodle или другую LMS как SCORM 1.2-материал.

Важно: каталог `api/_runtime_assets` используется для временных изображений и SCORM-архивов. При новой обработке старые runtime-материалы очищаются, поэтому не используйте этот каталог как постоянное хранилище.

## API

Backend предоставляет следующие основные endpoint'ы:

- `GET /formats` - список поддерживаемых расширений.
- `GET /llm/status` - информация о выбранном LLM-провайдере и наличии ключа.
- `POST /parse` - разбор файла без сборки лонгрида.
- `POST /pipeline` - разбор файла и сборка лонгрида.
- `POST /export/scorm` - сборка лонгрида и возврат SCORM ZIP.
- `GET /media?path=...` - выдача изображений из runtime-каталога.

### Лонгрид получается слишком "нечеловеческим" и нечитаемым

Скорее всего, backend работает в эвристическом режиме. 
Для LLM-анализа задайте `DEEPSEEK_API_KEY`.
